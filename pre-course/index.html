Pre-course

31 August 2015

So, I joined the Makers Academy September cohort.

We start on September 1st, but before that we have to finish the Pre-course. We are well equipped with several tools to go through the pre-course:

  • MA lessons,
  • Slack (where we can help each other)
  • and Google (where strangers can help us)!
Week 1: Command line

Command line interface (CLI) allows you to communicate directly with your computer, with underlying code of your computer, to be precise. You do not need support of graphical user interface (GUI) to create new files and folders, move between the folders and see what's inside, change access to files, to find stuff, etc.

Changing GUI to CLI feels a bit like removing training wheels form a bicycle. At first it is uncomfortable but later you appreciate the freedom you get.

Maker's lesson on CL is very informative and clear, very easy to read and understand even for a newbie (like myself!).

New words learned: pwd, ls, cd, mkdir, rmdir, rm, touch, cp, mv (renaming a file is actually moving a file!), cat, less, chmod, man, find, grep, sudo, shebang and many many others. Also learned how to use | < > << to redirect streams.

So with all these knowledge and trembling hands I successfully finished the first pre-course challenge. Only 3 weeks later I can say that it was just a kid's stuff...

Week 2: Git and Github

Week 2 started on Friday! I had a trip planned for the whole week 2, so had to finish my learnings for the week before it even has started.

Learning git is like receiving an access to a new universe. Because Github is a new universe! Populated with developers only.

Git is a version control system. And you cannot overrate the importance of using it. It helps you to save you work on different stages, to work on different versions of a program, to work on a project with a team, etc. You have to develop a habit to use it but it pays off.

The best thing is GitHub! It is like a huge open library of source code. You can keep your code there, but you can also browse through other developers code, fork it, clone it, even improve it (well, I am soooooo far from that now).

As a developer you are your Github profile. Feeling exposed!

Weeks 3: Ruby

From the beginning I thought that making mistakes and asking for help could be the most difficult things for me on this course.

But Makers are so good in helping you to ask for help!
There is Slack, where you can communicate with your cohort and seniors, there are Github issues, where you can ask questions, and on week 3 of pre-course you get a mentor! No way you are left on your own!

When you start learning actual programming language you need help more and more.

Ruby is a very popular programming language and a very elegant one! The more you learn it the more you appreciate how smart it is. My vocabulary expanded dramatically with words like object, class, method, variable, string, array, hash and tonns and tonns of method names...

This week I finished Chris Pine book, finished sections 1 and 2 of Ruby Kickstart, and (the most exciting) started to play on codewars!

Codewars are addictive. I have an awful mess at home, barely walk with my dog, and my partner is getting used to this absent look, when I pretend to participate in a conversation but actually can not stop thinking about an unresolved kata... you know...

At some point I forced myself to stick to 7 and 8 kyu kata, to learn how to write more elegant 'ruby style' code. Solving a problem is one thing but solving it with a nice code is so much more satisfying! My favourite trick now is to write if...else..end sentence in one line with <condition> ? <if true> : <if false>. It makes a code look much lighter!

So this week's challenge to reach 70 points on codewars was more fun then challenge! Stackoverflow helps a lot, of course.

During this week I also had my first session of pair programming with my tutor. Loved it!

Oh! And I also learned to use some Markdown to make my page look nicer!

Week 4: More Ruby

This week I was in Edinburgh for Fringe festival! Watched tonnes of great shows! And I must say that technology is everywhere, even in the opera! And of course in theatre!

My learning this week was between the shows, but 4 hour train to Edinburgh and back helped a lot. I covered most part of the Well-Grounded Rubyist by David A. Black. The book is so good! A real page-turner. It is also very smart for the publisher to give you a free ebook (password protected) when you buy a hardcopy.

Also I did some more katas and started reading about rspec.

And I spent my last day before the course writing this post! The fun begins in 15 hours!

Yana Proskurina
