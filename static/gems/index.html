Gems

13 September 2015

Makers' pill on gems is very good. So I will write here in few words what I read there.

Gems are ruby libraries that can be easily reused. There are lots of ruby gems.

You can install ruby gem using a command gem install <gem_name>.

But for real developers there is a better way! Bundler!

Bundler is a gem itself, and its purpose is to manage other gems (to install it write gem install bundler in the command line).

To tell bundler what gems are needed for particular program, Gemfile should be created in the root directory of the project. Gemfile contains this self-explaining line source 'https://rubygems.org' and a list of gems.

To use a gem in your code you should require gem, like this for example: 

require 'dotenv'

Gems are great, cause you can use other people's code. But they are not so great because of the same reason.

Yana Proskurina
