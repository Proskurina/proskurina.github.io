Days 35-37 Mainly MEAN

07 October 2015

Well, obviously I do not follow my plan...

At least I do not post here every day, but that is because I learn so many things simultaneously!

New knowledge for the last 3 days in few words:

MEAN stays for Mongo, Express, Angular and Node.

Node helps you to run your JS program using terminal, just print node <filename.js> in command line. If you type node in terminal you can play with JS the same way as you play with Ruby using irb (but I already got used to console and console.log()). It is not a server, but you can create one with it. And everything is in JS! So no need to switch from JS to Ruby (or another language) and back. An important thing about Node is that it is asynchronous, which makes it very effective. But you have to use callbacks for that, and it is not so straight forward in the beginning.

Express is something like Sinatra, but for Node JS. It seems easier and more intuitive, but may be it is because we already learned how to deal with Sinatra. For express template engine is not needed, it can handle html files directly, it is good for simple projects but for more complicated you can use Jade or ejs (they are like erb for ruby).

Mongo is a non SQL database, it stores information as JSON, and it seems a bit chaotic, but may be it is not, I just started to play with it.

And Angular... I am not sure yet. We will have the whole week dedicated to it, so I will write about it later.
