22 October 2015

This is a short addition to my previous post.

So, you are deploying your app to heroku.

But you do not want to push bower_components folder to github or heroku (so bower_components is in your .gitignore file).

How to make heroku run bower install and install bower_components to the right folder?

3 simple steps:

1) Include bower as dependency into your package.json file like this: 

  "dependencies": {
    "bower": "^1.6.3"
  }

It will tell heroku to install bower in the first place.

2) In the same package.json file tell heroku to run bower install. You do it like this:

  "scripts": {
    "postinstall": "bower install"
  },

3) Indicate where you want bower_components to be installed. By default this folder will appear in your root directory, if you want it to be in another place you need .bowerrc file.

For example, if you want it to be in public folder you indicate it in your .bowerrc file:

{
  "directory" : "public/bower_components"
}

Now heroku knows what to do!
