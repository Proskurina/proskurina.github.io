Today we have to ​describe the difference between composition and inheritance.

Well, Zed A. Shaw explained it nicely and clearly here. I do not think I can do better!

What are some of the dangers of inheritance?

First, you can inherit only from one class, so it is important to choose carefully.
And of course it is important to apply logic here, as though a plane and a bike might have some similarities (take you from point 1 to point be, can be broken, etc.), it does not mean that logically Bike can be a sub-class of a Plane or vice versa.
It can be hard to test classes that reuse code from a super-class (everything about super class or super-super class should be at hand).

Why would we prefer composition over inheritance?

We can include as many modules to a class as we want, not just one. And they are easy to implement (no additional difficulties comparing to classes)! We can avoid a lot of repetition with them (DRY principle!).

It is also possible to do composition using other classes but it is more complicated then with modules.

And below is some real life example of class inheritance!

class Ship

  attr_reader :position, :direction, :size, :hits

  def initialize(position, direction)
    @position = position
    @direction = direction
    @size = 1
    @hits = 0
  end

  def was_hit
    @hits += 1
  end

  def sunk?
    @hits == @size
  end

end

class Submarine < Ship

  def initialize(position, direction)
    super(position, direction)
    @size = 2
  end

end

And when you print this: 

sub = Submarine.new('C3', :E)  
p sub.size  
p sub.hits

You get this: 

2  
0
